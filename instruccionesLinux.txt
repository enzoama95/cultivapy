Pasos para implementar cultiva.py en un Linux de cero

Checkear version de Python
    python3 --version
    Deberia verse algo asi: "Python 3.8.5"

Install postgres
    sudo apt-get install postgresql

Cambiar el password del usuario de postgres (https://chartio.com/resources/tutorials/how-to-set-the-default-user-password-in-postgresql/)
    Iniciar sesion en postgres como root
        sudo -u postgres psql
    Poner una nueva contraseña
        ALTER USER postgres PASSWORD 'miNuevaContraseña';


Permitir al usuario no root acceder a Postgres
    Modificar el archivo de configuracion pg_hba.conf para cambiar el parametro "peer" a "md5"
        sudo nano /etc/postgresql/12/main/pg_hba.conf 
            reemplazar todos los "peer" por "md5" y guardar el archivo
            

    Reiniciar servicio postgres
        sudo service postgresql restart

Probar que funciona
    psql -U postgres -W 
    poner la contraseña
    Se debe ver: " psql (12.6 (Ubuntu 12.6-0ubuntu0.20.04.1))
                   Type "help" for help.
                   postgres=#
                 "
    Listo! Para salir, usar "\q" y enter

Instalar postgis
    sudo apt-get install postgis postgresql-12-postgis-3


Instalar gettext
    sudo apt-get install gettext

Instalar Python3 venv
    sudo apt-get install python3-venv

Clonar el repositorio de cultiva.py desde github
    git clone git@github.com:enzoama95/cultivapy.git

Cuando tengas el repositorio de forma local, ejecutar los siguientes comandos desde la terminal:
    cd cultivapy
    python3 -m venv cultivapy-env
    source cultivapy-env/bin/activate
    pip install --upgrade pip
    pip install -r requirements.txt
    cp conf/.env.example conf/.env # Debes editar la conf de ".env" con tus propios datos como indica el "env.example"
    
Una vez modificado el ".env" con los datos a utilizar, hay que crear una database con el nombre utilizado en ".env" file
    Iniciar sesion en postgres
        psql -U postgres -W
    Crear BD llamada "cultivapy_db"
        create database cultivapy_db;

Cuando tengamos lista la base de datos, hacemos la migración, compilación de mensajes y ponemos a correr el servidor:
    ./manage.py migrate
    ./manage.py compilemessages
    ./manage.py runserver
